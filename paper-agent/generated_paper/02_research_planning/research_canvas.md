# Research Canvas

| 要素 | 内容 |
|:---|:---|
| **1. 研究テーマ** | コードAGENTを用いた市民開発におけるセキュリティリスクの構造化と多層ガバナンスモデルの提案 |
| **2. 解決したい課題** | プログラム知識が低い市民開発者が、コードAGENT生成物に含まれるセキュリティ脆弱性を認識・検出できず、個人情報漏洩や業務停止を引き起こす本番デプロイリスク。 |
| **3. なぜ今重要か** | 市民開発の急速な普及とLLMコードエージェントの浸透により、シャドーITや脆弱コードの爆発的増加が懸念されるため。既存のSASTツールや高度エンジニア前提のレビュー体制は市民開発者には適用不能。 |
| **4. Research Questions** | **RQ1**: 市民開発環境でのコードAGENT生成物における脆弱性発生傾向と既存ガバナンスの限界は何か？<br>**RQ2**: マルチエージェント自動検証とプロセスガードレールを融合した多層ガバナンスモデルは、開発容易性を維持しつつ脆弱性を抑止できるか？ |
| **5. 仮説 (Hypotheses)** | **H1**: コードAGENTは機能実現を優先し、セキュリティ検証なしでは特定脆弱性を高頻度で含む。<br>**H2**: 自動検証マルチエージェントと非エンジニア対応プロセスガードレールの統合により、脆弱性混入率を有意に低下させられる。 |
| **6. 提案アプローチ** | **技術的制御**: 生成エージェント＋専用セキュリティ検証/修正エージェントの構成。<br>**組織プロセス制御**: 知識不要の自動判定・要約ガードレール（デプロイ前承認ゲート）。 |
| **7. 評価方法** | **A. プロトタイプ実験**: 開発タスクに対する脆弱性抑止率・自動修復率の測定。<br>**C. ケースシナリオ分析**: 大学図書館・研究機関における運用シナリオでの実行可能性評価。 |
| **8. 期待される貢献** | 1. LLM市民開発におけるセキュリティリスクタクソノミーの策定。<br>2. 技術×プロセス統合型の多層ガバナンスフレームワークの構築。<br>3. 研究機関・大学等での安全なAI活用モデルの提供。 |
| **9. 投稿先** | 大学図書館研究（研究機関の研究報告・AI特集） |
