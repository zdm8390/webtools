# リサーチギャップ抽出レポート (Research Gap Analysis)

## 抽出された5つのリサーチギャップ

### 1. Target Gap（対象者のミスマッチ）
- **既存研究**: 主に「高度なプログラマ・システムエンジニア」を対象としてAIコード生成ツールの影響や脆弱性を分析している（Sandoval et al., 2023 等）。
- **ギャップ**: プログラミング知識を持たない/乏しい「市民開発者（非IT部門職員）」がAIコードAgentを使った場合に発生する固有のリスク行動や知識非対称性に関する研究が不足している。

### 2. Methodological Gap（ガバナンスアプローチの欠如）
- **既存研究**: AI生成コードの「脆弱性発見率の定量測定」に焦点が当てられている（Pearce et al., 2022 等）。
- **ギャップ**: 脆弱性を発見した後に「非専門家開発者のフローをどう制御し、修正まで自動誘導・制御するか」という実用的なガバナンスアプローチが提案されていない。

### 3. Usability Gap（既存SASTツールの適用限界）
- **既存研究**: 静的解析ツール（SAST）や従来のコードレビューの検知限界を評価している（Asare et al., 2023）。
- **ギャップ**: 抽象的なSASTの警告ログを市民開発者が解釈できない問題に対し、自然言語によるリスク要約や修正代行（マルチエージェント修復）を組み合わせるアプローチが未検証である。

### 4. Integration Gap（技術と組織プロセスの分断）
- **既存研究**: 「AI技術単体（LLMエージェント）」の研究か、「組織ルール・ガイドライン（OWASP/Gartner等）」の研究のどちらか一方に閉じている。
- **ギャップ**: マルチエージェントによる技術的自動検証と、デプロイ前の組織的プロセスガードレールを「ひとつの物理システム・構造」として統合した多層モデルが存在しない。

### 5. Context Gap（特定組織領域における検証不足）
- **既存研究**: 一般企業の商業開発やオープンソースコミュニティを対象とする。
- **ギャップ**: 大学図書館や研究機関等、個人情報・研究データを扱う一方でIT専門人員が不足している公共・学術組織コンテキストにおける市民開発ガバナンスモデルの適用可能性検証が不足している。
