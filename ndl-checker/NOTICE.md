# NDL公開状況チェッカー 利用上の注意事項

本ツール（NDL公開状況チェッカー）を利用するにあたり、以下の事項について事前にご確認ください。

---

## 1. NDL Search APIの利用規約と申請について
本ツールは、国立国会図書館が提供する「国立国会図書館サーチ（NDL Search）OpenSearch API」を利用しています。

* **非営利かつ個人での利用**：
  * 事前の利用申請は不要で、自由に検索をご利用いただけます。
* **営利目的・大量アクセスでの利用**：
  * 本ツールを商用サービスに組み込む、または非営利であってもスクレイピングやクローラー等を用いて継続的・大量のアクセスを行う場合は、**国立国会図書館への事前の「API利用申請」が義務付けられています**。
  * 詳細および申請方法については、国立国会図書館サーチ公式の「[APIのご利用について](https://ndlsearch.ndl.go.jp/help/api)」をご確認ください。

---

## 2. メタデータの二次利用について
検索結果として表示される書誌情報や公開条件などのメタデータは、複数の機関や出版社などの「データプロバイダ（提供元）」から収集されたものです。

* メタデータの著作権やライセンス（二次利用のルール）は、資料および提供元ごとに異なります。
* データを抽出し、別の用途で二次利用（転載やデータベース化）する場合は、必ず各資料レコードの権利情報（`dc:rights`）を確認し、提供元のポリシーに従ってください。

---

## 3. CORSプロキシ（外部サービス）への依存について
国立国会図書館サーチのAPIは、ブラウザから直接JavaScriptでアクセスする「CORS（Origin間リソース共有）」に対応していません。

* 本ツールは、クライアントサイド（ブラウザ）のみでCORS制限を回避するため、パブリックなCORSプロキシサーバー（`https://api.allorigins.win/`）を経由してAPIリクエストを中継しています。
* そのため、**中継プロキシサーバーの一時的な障害、停止、または応答速度低下が発生した場合、本ツールでの検索も一時的に利用できなくなるか、動作が遅くなる場合があります**。あらかじめご了承ください。

---

## 4. 公開判定の正確性について
本ツールは、APIから返却されるXMLデータ内の特定の項目（`rights`, `accessRights`, `description` など）に含まれるキーワード（「インターネット公開」「送信サービス」「館内限定」等）を機械的にスキャンし、簡易判定を行っています。

* メタデータの記述形式のゆらぎやデータ更新のタイムラグにより、実際の国会図書館デジタルコレクションでの公開制限と異なる判定結果が表示される場合があります。
* 正確な最新の公開状況や閲覧可否については、必ず検索結果カードから遷移できる**「国立国会図書館デジタルコレクション（dl.ndl.go.jp）」の公式ページにて直接ご確認ください**。

---

## 5. 複数一括検索におけるレートリミット制御について
本ツールには、改行区切りで複数のキーワードを同時に検索できる「複数一括検索モード」が搭載されています。国会図書館サーチのサーバーに過度な負荷（DDoS攻撃とみなされるアクセス）をかけないため、以下のアクセス制御を行っています。

* **行数制限（最大10件）**：一度に同時入力・検索できるキーワード数は最大10件に制限されています。10件を超えた場合は、最初の10件のみが実行対象となります。
* **逐次実行とウェイト（遅延）の挿入**：複数のリクエストを並列で一気に送信するのではなく、1キーワードずつ順番に処理を行い、各APIリクエストの間に**「約0.6秒（600ms）のウェイト（待機時間）」**を自動で挿入しています。
* 連続して一括検索を行う場合は、過剰な負荷を避けるため、一定の間隔をあけてご利用いただきますようご協力をお願いいたします。

---

## 6. カスタムCORSプロキシ（Google Apps Script）の設定方法
国会図書館（NDL）本体のAPIはブラウザからの直接アクセス（CORS）を拒否しているため、外部のプロキシサーバーを経由する必要があります。しかし、パブリックな無料プロキシ（Alloriginsなど）は海外IPからのアクセスのため、NDL側のセキュリティファイアウォール（WAF）にブロックされてタイムアウトや接続エラー（403/408）になりがちです。

本ツールでは、自分専用の **Google Apps Script (GAS) CORSプロキシ** を設置して利用することができます。Googleの安定した日本国内サーバーを経由するため、NDLにブロックされることなく安全・爆速（0.1秒以下）で通信可能です。

### 🛠️ GASプロキシの設置手順（所要時間3分）

1. **Google Apps Scriptエディタを開く**
   * [Google ドライブ](https://drive.google.com/) にアクセスし、左上の「新規」 ＞ 「その他」 ＞ 「Google Apps Script」 をクリックします。
   * プロジェクト名（画面左上）を「NDL-CORS-Proxy」など分かりやすい名前に変更します。

2. **コードを貼り付ける**
   * 最初から入っている `myFunction` の記述をすべて消去し、以下のコードをそのままコピー＆ペーストします。

   ```javascript
   function doGet(e) {
     var targetUrl = e.parameter.url;
     if (!targetUrl) {
       return ContentService.createTextOutput("Error: Missing url parameter").setMimeType(ContentService.MimeType.TEXT);
     }
     try {
       // 指定されたNDLのURLにアクセスしてデータを取得
       var response = UrlFetchApp.fetch(targetUrl, { muteHttpExceptions: true });
       var text = response.getContentText("UTF-8");
       
       // CORS（ブラウザ直接アクセス）を許可するヘッダーを付与して返す
       return ContentService.createTextOutput(text)
         .setMimeType(ContentService.MimeType.TEXT)
         .setHeader("Access-Control-Allow-Origin", "*");
     } catch(err) {
       return ContentService.createTextOutput("Error: " + err.toString())
         .setMimeType(ContentService.MimeType.TEXT)
         .setHeader("Access-Control-Allow-Origin", "*");
     }
   }
   ```

3. **ウェブアプリとしてデプロイする**
   * 画面右上の青い「**デプロイ**」ボタン ＞ 「**新しいデプロイ**」 をクリックします。
   * 左上の歯車アイコン（種類の選択）をクリックし、「**ウェブアプリ**」 を選択します。
   * 設定項目を以下のように指定します：
     * **次のユーザーとして実行**: `自分`
     * **アクセスできるユーザー**: `全員`
   * 下部の「**デプロイ**」ボタンをクリックします。
   * 必要に応じて「アクセスを承認」を求められた場合は、アカウントを選択して許可してください。

4. **URLのコピーとツールへの設定**
   * デプロイ完了画面に表示される「**ウェブアプリのURL**」（`https://script.google.com/macros/s/xxxx/exec`）をコピーします。
   * 本ツールの画面上部にある「**カスタムプロキシ**」入力欄に貼り付けます。
   * ※入力したURLはブラウザ（localStorage）に自動保存され、次回起動時も自動的にセットされます。
