# 指示書: Vampire Survivors風ブラウザゲームの開発（GitHub Pages対応版）

あなたは優秀なゲームエンジニアおよびフロントエンド開発者です。
HTML5 CanvasとJavaScript（Vanilla JS）を使用し、ブラウザ上で軽快に動作するヴァンパイア・サバイバーズ風2Dアクションゲームを構築してください。
本プロジェクトはGitHub Actionsを用いてGitHub Pagesへ自動デプロイされる前提の構成とします。

---

## 1. プロジェクト構成

外部のビルドツール（Webpack等）やアセット画像ファイルに依存せず、ブラウザ標準機能のみで動作するシンプルな構成とします。

* `index.html`（HTML構造、UI要素、Canvas配置）
* `style.css`（全画面表示、Canvas中央配置、UIレイアウト）
* `game.js`（ゲームループ、オブジェクト指向による状態管理・描画・ロジック）
* `.github/workflows/deploy.yml`（GitHub Pages自動デプロイ用ワークフロー）

---

## 2. ゲームバランスと演出

* **プレイ時間制限**: 1回のプレイ時間は最大10分間（600秒）。画面上部に残り時間をカウントダウン表示する。
* **成長の爽快感**: レベルアップ時のステータス上昇（攻撃力、攻撃範囲、移動速度、攻撃頻度）の体感を強くし、段階的に画面全体を攻撃できるような爽快感を重視する。
* **ダメージ表示**: 敵に攻撃がヒットした際、ヒットした敵の位置に与えたダメージ数値（黄〜赤色のテキスト）を出現させ、上部にふわっと浮き上がりながら消えるエフェクト（ダメージポップアップ）を実装すること。

---

## 3. グラフィックと描画処理

動作の軽量化を図るため、画像ファイルの代わりにCanvasのテキスト描画機能（`ctx.fillText`）を用いて文字や絵文字（Unicode）をグラフィックとして使用してください。

* **プレイヤー**: 🗡️ や 🧙‍♂️ などのアイコン
* **敵キャラ**: 
  * ザコ敵: 👾 や 🦇（移動速度は速いがHPは低い）
  * 中ボス: 💀 や 👹（移動速度は遅いがHPが高い）
* **経験値ジェム**: 💎 や ✨
* **攻撃エフェクト**: ✦ や 💥 などの記号または線描画

---

## 4. コア機能仕様

### 4.1. プレイヤーおよび操作
* `W/A/S/D` または 矢印キーで移動する。
* 移動速度、最大HP、現在HP、獲得経験値、レベルのステータスを保持する。

### 4.2. 敵の生成と難易度上昇
* 画面外から敵が継続的にスポーンし、プレイヤーに向かって追従する。
* 経過時間に伴い、同時スポーン数、敵のHP、移動速度が段階的に上昇する。

### 4.3. 自動攻撃システム
* タイマー制御により、最寄りの敵に対して自動的に攻撃を射出・発生させる。
* レベルアップに応じて「攻撃力」「射程・範囲」「発射数」「攻撃間隔」が強化される。

### 4.4. 経験値・レベルアップ
* 敵撃破時にその場に経験値アイテムをドロップする。
* プレイヤーが接近すると自動で吸引される。
* レベルアップ時、ゲームを一時停止し、3つのランダムな強化案（ビルド選択）をUI表示する。選択後、ゲームを再開する。

### 4.5. ハイスコア機能（ランキング）
* `localStorage` を使用し、過去の最高スコア（撃破数または獲得スコア）上位5件を保持・更新する。
* ゲームオーバー時およびタイトル画面に「ハイスコアランキング」を表示する機能を含めること。

---

## 5. CI/CD設定

* `.github/workflows/deploy.yml` を作成し、`main` ブランチへの push 時に GitHub Pages へ静的ファイルを自動デプロイするワークフローを記述すること（`actions/deploy-pages` を使用）。
