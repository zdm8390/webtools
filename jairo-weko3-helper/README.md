# JAIRO-Cloud / WEKO3 リポジトリ登録支援ツール

本ツールは、大学図書館等の学術機関リポジトリ（JAIRO-Cloud / WEKO3）における論文登録業務を効率化するためのツール群です。  
同一のコアロジックを搭載した **「Chrome拡張機能 (Manifest V3)」** と、環境制限のある端末向けの **「ブックマークレット」** の2通りを提供します。

---

## 📁 ツール構成とファイル役割

本パッケージは以下のファイルで構成されています。

* **[manifest.json](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/manifest.json)**: Chrome拡張機能の定義ファイル（Manifest V3）。アクティブタブの読み取りおよび動的スクリプト実行のための最小権限を設定しています。
* **[content.js](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/content.js)**: アクティブなWebページ上で動作するDOM解析スクリプト。選択したテキスト、または `<head>` 内のメタタグからDOIを探し出します。
* **[popup.html](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/popup.html)**: 拡張機能のポップアップ画面。直感的で美しいガラスモルフィズム風ダークテーマを採用し、取得データの編集・確認が可能です。
* **[popup.js](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/popup.js)**: ポップアップの動作制御。Crossref APIとの非同期通信、著作権ポリシーの簡易チェック、CSV/TSV生成を実行します。
* **[bookmarklet.js](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/bookmarklet.js)**: ブックマークレット用のコード。ブラウザへ登録するための「1行（Minify）」コードと、開発・カスタマイズ用の「可読」コードを両方含んでいます。

## 🚀 導入方法 (Installation)

### A. Chrome拡張機能のインストール手順
拡張機能のインストールは非常にシンプルで、管理者権限のないPCでもChromeの「デベロッパーモード」を使用すれば数分で導入可能です。

1. Google Chromeを起動し、アドレスバーに `chrome://extensions/` を入力して開きます。
2. 画面右上にある **「デベロッパー モード」** のトグルスイッチをONにします。
3. 画面左上に表示される **「パッケージ化されていない拡張機能を読み込む」** ボタンをクリックします。
4. 本フォルダ `jairo-weko3-helper` （`manifest.json` が入っているディレクトリ）を選択します。
5. インストールが完了すると、ツールバーのパズルピースアイコン（拡張機能メニュー）から「JAIRO-Cloud / WEKO3 リポジトリ登録支援ツール」を選択し、ピン留めして利用可能になります。

---

### B. ブックマークレットの登録手順
事務用PCなどでChromeの拡張機能追加すら制限されている環境では、以下のブックマークレット機能で代用します。

1. **[bookmarklet.js](file:///C:/Users/hana/Documents/GitHub/webtools/jairo-weko3-helper/bookmarklet.js)** を開き、上部の `/* ... */` コメント内に記載されている `javascript:(async...` から始まる**コードをすべてコピー**します。
2. ブラウザで任意のページをブックマークに登録します（または `Ctrl + D` で適当なページを登録）。
3. 登録したブックマークを右クリックして **「編集」** または **「プロパティ」** を開きます。
4. 設定を以下のように編集し、保存します。
   * **名前**: `📖 WEKO3 Importer` (判別しやすい任意の名前)
   * **URL / アドレス**: コピーした `javascript:...` から始まるコードをそのまま貼り付けます。

## 📖 使用方法 (How to Use)

### 💡 共通の「DOIハイブリッド検出」仕様について
本ツールは画面上の全自動スキャンによるノイズ（誤検出）を防ぐため、以下の優先アルゴリズムに基づいて動作します。
1. **優先度1 (選択範囲)**: マウスで論文上のDOI文字列（例: `10.1038/...`）を選択ドラッグしている場合、その選択テキストからDOIを取り出します。
2. **優先度2 (メタタグの自動走査)**: テキストが選択されていない場合、開いているタブのHTML内（`citation_doi`, `dc.identifier`, `prism.doi` 等）から自動でDOIを抽出します。
3. **優先度3 (手動入力)**: 自動抽出できなかった場合、手入力用のフィールドが表示（または入力プロンプトが起動）され、手動での処理が可能です。

---

### 1. Chrome拡張機能での登録手順
1. CiNii Research、Nature、PubMed、IEEE Xploreなどの論文詳細ページを開きます。
2. （必要に応じて）画面上のDOIテキストをマウスでドラッグ選択します。
3. 拡張機能のアイコンをクリックしてポップアップを開きます。
4. **「検出完了」** と表示されているのを確認し、**「Crossrefから書誌メタデータを取得」** をクリックします。
   * 自動検出されなかった場合は、入力欄に直接DOIを入れてください。
5. 取得に成功すると、書誌メタデータがフォームに入力されて展開されます。内容に誤りや不足があれば、**その場で直接編集可能**です。
6. セルフアーカイブポリシー判定（SCPJ/Sherpa Romeoリンク）を確認します。
7. 「CSV」または「TSV」形式を選択し、**「インポート用データ出力」** をクリックします。
   * Excelで開いた際の文字化けを防ぐ **BOM付きUTF-8** でファイルが自動ダウンロードされます。
8. ダウンロードが完了すると、ファイルがローカルに保存されます。

---

### 2. ブックマークレットでの登録手順
1. 対象の論文ページを開きます（必要に応じてDOIを選択しておきます）。
2. ブックマークバーに登録した `📖 WEKO3 Importer` をクリックします。
3. 自動検出された場合はそのまま進行します。検出されなかった場合はダイアログが表示されますので、DOIをコピペして「OK」をクリックします。
4. 画面右上に「Crossref API 連携中...」と小さくトーストが表示され、APIから書誌データを取得します。
5. 取得・解析が完了すると、自動的にWEKO3インポート用CSVファイルがローカルにダウンロードされます。
6. 同時に、画面中央に美しい半透明の**完了通知モーダル**が表示されます。
   * モーダル上で取得されたタイトル、ISSNによる著作権ポリシーへの確認リンクを確認し、確認後は「閉じる」で終了します。

## 📥 WEKO3 (JAIRO-Cloud) へのインポート手順
ダウンロードしたCSV/TSVファイルは、リポジトリ管理者権限で以下の手順で一括取り込みが可能です。

1. **JAIRO-Cloud / WEKO3 管理画面**にログインします。
2. 左メニューより **「アイテム管理」** ＞ **「インポート」** を選択します。
3. インポート画面にて、エクスポートした CSV または TSV ファイルをドラッグ＆ドロップまたはファイル参照にて選択します。
4. ファイルのエンコーディングが「UTF-8 (BOM付き)」であることを確認します。
5. **「インポートのプレビュー」** を実行し、ヘッダーマッピング（1行目・2行目・3行目の指定など）が正しく認識され、書誌情報が正しくセルに展開されているか確認します。
   * 本ツールの出力はJPCOARスキーマに対応しているため、基本的には自動でマッピングが完了します。
6. プレビューに問題がなければ、**「インポート実行」** ボタンをクリックします。

## 🛠️ トラブルシューティング (Troubleshooting)

| 症状 | 推定原因 | 対処方法 |
| :--- | :--- | :--- |
| **ブックマークレットをクリックしても何も動かない、またはエラーダイアログが出る** | **CSP (Content Security Policy) による通信制限** <br>親サイト（例: GitHubなど）のセキュリティ設定によって、外部ドメインへのfetch通信が遮断されています。 | ブラウザのセキュリティ仕様による制限です。通信制限を受けない **「Chrome拡張機能版」** を使用してください。 |
| **ダウンロードしたCSVをExcelで開くと文字化けする** | **エンコーディングの不一致** <br>ExcelがファイルにBOM（Byte Order Mark）が含まれていないと判断したか、文字コード設定が異なっています。 | 本ツールはExcel用対策としてBOM付きUTF-8でファイルを出力しています。万が一文字化けする場合は、Excelの「データ」タブ＞「テキストファイルから」から文字コード「UTF-8」を指定して取り込んでください。 |
| **DOIが「未検出」になる** | **ページ上にDOIメタデータが存在しない** <br>論文以外のページを開いているか、そのWebサイトが標準的なメタタグをサポートしていません。 | 論文上のDOI文字列（`10.xxxx/xxxx`）をマウスでドラッグ選択した状態で、再度スキャンを実行するか、手動入力エリアに入力してください。 |
| **iframe内のDOIが検出できない** | **クロスドメインセキュリティ制限** <br>PDFプレビュー画面などが `iframe`（埋め込み）で表示されている場合、親スクリプトからフレーム内の選択が取得できません。 | PDF表示画面の外側（書誌情報が書いてあるHTML側）のテキストを選択してスキャンするか、メタデータからの自動取得に頼るか、手動で入力してください。 |
