# システム要件定義書および実装仕様書：『テトリスン』

## 1. プロジェクト概要

* **タイトル**: テトリスン
* **プラットフォーム**: Webブラウザ（GitHub Pagesによる静的ホスティング対応）
* **ターゲット動作環境**: PCブラウザ（レスポンシブ対応、アスペクト比 16:9 固定スケーリング）
* **操作方式**: キーボード（方向キー、A/S、Space）およびゲームコントローラー（Gamepad API）
* **世界観・デザイン方針**:
  * 魔法が存在する世界の動物たちによるテトリス対戦。
  * 白を基調としたパステルカラー、角が丸いPOPなテトリミノ。
  * 爽快感を意識したエフェクト（ライン消去時の画面振動、キャラ固有演出）。
  * 全画面がスクロールなしの1画面内に収まるレスポンシブUI構成。

---

## 2. 技術スタックおよびシステム構成

* **フロントエンド**: HTML5, CSS3, Pure JavaScript (ES6+) または TypeScript
* **描画ライブラリ**: HTML5 Canvas API（パフォーマンス向上とエフェクト制御のため）
* **シナリオデータ形式**: YAML（外部ファイルとして分離・編集可能）
* **データ保存**: `localStorage`（ハイスコア、キーコンフィグ、設定情報等）
* **オンライン対戦／乱入機能**: WebRTC (PeerJS 等) による P2P 接続（サーバーレスで GitHub Pages 上で完結する構成）
* **国際化・テキスト制御**: 日本語表示時、指定文節での適切な改行位置チェックおよび自動折り返し制御ロジックを実装。

## 4. ゲームモード詳細

### 4.1 ストーリーモード
* **構成**: 難易度選択に応じたステージ進行。
  * **Easy**: 3ステージ
  * **Normal**: 5ステージ
  * **Hard**: 7ステージ
* **会話劇システム**:
  * 各ステージ開始前にYAMLファイルから読み込んだ対話イベントを再生。
  * キャラクター同士の関係性を表現するアドベンチャーパート画面。

### 4.2 シングル対戦モード
* **形式**: CPUとの1対1対戦。
* **難易度**: 思考速度およびブロック設置精度を変更可能。
* **乱入機能**: 乱入ONの場合、P2P接続を待受し、他プレイヤーの参加要求があった場合にオンライン対戦へ切替可能。

### 4.3 スコアアタックモード
* **形式**: シングルプレイのエンドレスモード。
* **速度仕様**: 消去ライン数または経過時間に伴い、落下速度（Gravity）が段階的に上昇。

## 5. ゲームシステム・ルール詳細

### 5.1 基本対戦・お邪魔ラインルール
* **お邪魔ライン送出数（標準ぷよぷよ/テトリス互換基準）**:
  * 1ライン消去: 0行
  * 2ライン同時消去: 1行
  * 3ライン同時消去: 2行
  * 4ライン（テトリス）同時消去: 4行
  * REN（連続消去）ボーナス: 1REN毎に追加+1行
* **お邪魔ラインの保留・相殺（ぷよぷよ方式）**:
  * 受信したお邪魔ラインは直ちに盤面に出現せず、「予告予告ゲージ」にストックされる。
  * 自分がラインを消去してお邪魔ラインを送出する際、まず自分の予告ゲージにあるお邪魔ラインと**相殺**される。
  * 相殺しきれなかったお邪魔ラインは、自分のテトリミノが接地・固定されたタイミングで盤面下部からせり上がる。

### 5.2 魔法システム
* **ゲージ蓄積条件**: 経過時間（毎秒微量蓄積）＋ お邪魔ラインの被弾（受け取った行数に応じて大きく蓄積）。
* **発動メカニズム**:
  1. ゲージ満タン時にプレイヤの任意入力（Spaceキー / 対応ボタン）で魔法ルーレットが起動。
  2. ルーレットによりLV0〜LV4の効果が決定し、即時発動する。
* **ルーレット効果表**:
  * **LV0**: はずれ（効果なし）
  * **LV1**: 自分の盤面の全お邪魔ラインを消去
  * **LV2**: 今後5カウントの間、相手からの追加お邪魔ラインが到着しない
  * **LV3**: 次に出現するテトリミノが5回連続で「I型（縦棒）」に固定化される
  * **LV4**: キャラクター固有魔法（後述）

## 6. キャラクターおよび独自魔法（LV4）仕様

対戦中、プレイヤーおよびCPUのグラフィック領域において、ダメージ（お邪魔被弾）時は悲しい顔、ライン消去時は喜ぶ顔等のコミカルな表情変化およびエフェクトアニメーションを表示する。

| キャラクター | 元動物 | LV4 独自魔法効果名称 | 具体的なゲーム内効果 |
| :--- | :--- | :--- | :--- |
| **ネコニャー** | 猫 | 気まぐれアタック | 自身の盤面の奇数行（1, 3, 5行目等）からランダムに3行分を消去する。 |
| **イヌワン** | 犬 | カエシテク・アタック | 現在自分の盤面にあるお邪魔行をそのまま相手に押し戻す（最大3行）。 |
| **トリチュン** | 鳥 | ピーチクテイル | 自身の盤面の最上段から3行分のブロックを消去する。 |
| **サルウッキー** | 猿 | ウッキッキー・スクエア | 次に出現するテトリミノが3回連続で「O型（四角）」に固定化される。 |
| **タヌポン** | タヌキ | ドロンコ・リセット | お互い（自分と相手）のステージ盤面を完全に空の初期状態へ戻す。 |
| **キツネコン** | 狐 | トリック・チェンジ | 自分の盤面状態と相手の盤面状態をそのまま入れ替える。 |
| **ドラゴゴン** | 竜 | ドラゴン・ブレス | 自身の盤面の下部から5行分のブロックを強力に消去する。 |

## 7. UI/UX・デザイン・演出要件

* **カラーパレット**: 白（#FFFFFF）および淡いパステルカラー（パステルピンク、パステルブルー、パステルイエロー等）をベースとする。
* **テトリミノデザイン**: 矩形の四隅に角丸（Border Radius相当）処理を施したポップなデザイン。
* **消去エフェクト**: ライン消去時に画面全体へのシェイク（振動）処理、および各キャラクターごとのエフェクト（猫パンチ、炎、羽等）をオーバーレイ表示。
* **レスポンシブ対応**:
  * アスペクト比 `16:9` を維持した Canvas / DOM の自動トランスフォーム（CSS `transform: scale()` 等）により、ウィンドウサイズに依存せず1画面内へ収まる構成とする。

## 8. セキュリティ・品質管理・検証要件

### 8.1 セキュリティ要件
1. **XSS（クロスサイトスクリプティング）対策**:
   * スコア登録時や対戦相手からの受信データにおいて、HTMLエスケープ処理（または `textContent` の使用）を徹底する。
2. **入力値検証（Validation）**:
   * 外部YAMLファイル解析時のスキーマ検証を実施。
   * WebSocket / P2P 通信で受信したゲームステートデータの異常値チェックを行う。

### 8.2 コード解析および検証手順
1. **静的解析**:
   * ESLint / TypeScript コンパイラ等による構文チェック・型チェックの実施。
2. **全体解像度・不具合チェック**:
   * プログラム修正後、全モジュール（描画系、物理/ロジック系、通信系、UI系）の相互作用による不具合がないか再解析を行う。
   * メモリリーク（`requestAnimationFrame` や イベントリスナーの解除漏れ）のチェックを実施する。
3.プロのコーダーの視点から不具合が無いか再度全プログラムをチェックする
